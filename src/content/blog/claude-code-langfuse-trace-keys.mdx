## 배경

Claude Code 를 다음과 같이 띄우면 내부 동작이 OTel span 형태로 흘러나옵니다.

```bash
claude --debug
```

OTel 은 OpenTelemetry 의 줄임말로, 한 프로그램의 동작을 trace / metric / log 형태로 외부 도구가 받아볼 수 있게 표준화한 observability 프레임워크입니다. trace 와 span 은 그 중 trace 부분의 핵심 두 단어인데, 둘의 관계는 다음과 같습니다.

이 stream 을 OTLP exporter 로 받아서 [Langfuse](https://langfuse.com) 에 보내면, 토큰 수, 시스템 프롬프트, tool 정의, 응답 본문, latency 까지 모두 위 트리의 각 노드에 담겨 시각화됩니다.

이 reference 는 그 trace 안의 모든 attribute 를 한 자리에 정리한 자료입니다. Claude Code 가 자체적으로 넣어주는 attribute 위에, Langfuse UI 가 잘 알아먹는 key 들 (`langfuse.observation.input/output`, `gen_ai.*`) 을 직접 추가했습니다. 그래서 Langfuse 에 도착한 한 span 에는 **Claude Code 가 원래 넣은 key** 와 **직접 추가한 key** 가 섞여 있고, 거기에 **OTel 표준 / OTel-GenAI / Langfuse 인식 key** 라는 출처 구분도 따로 있습니다.

정식 글로 풀어쓰기엔 너무 좁고 raw 한 내용이라 Lab Notes 로 둡니다. 본인 환경에서 Langfuse trace 를 봤을 때 "이 attribute 는 누가 넣은 거고 어디서 쓰이지?" 가 궁금할 때 들춰보면 됩니다.

## Attribute 카탈로그

trace export 는 세 그룹 (<span class="k">attributes.&#42;</span> / <span class="k">resourceAttributes.&#42;</span> / <span class="k">scope.&#42;</span>) 으로 들어옵니다. 이 중 의미 있는 데이터는 거의 전부 첫 번째인 <span class="k">attributes.&#42;</span> 에 모이고, 나머지 둘은 host/process 정보나 SDK 식별자처럼 자리에서 의미가 바로 드러나는 raw 한 값들입니다. 그래서 본 카탈로그에서는 <span class="k">attributes.&#42;</span> 만 다룹니다.

<span class="k">attributes.&#42;</span> 안의 key 는 어떤 span 위에 붙느냐에 따라 의미가 갈리므로, 아래 §1 ~ §5 는 span 종류별로 묶어 정리했습니다 (본문 안의 §3.1 / §3.4 같은 표기는 같은 카탈로그 안의 cross reference 입니다). 본체 표 앞에 보조 설명 두 개 (Trace 구조, Provenance) 를 먼저 둡니다.

### Trace 구조: 세 그룹

각 그룹에 구체적으로 어떤 값이 들어가는지 한 줄씩 짚으면:

- <strong><span class="k">attributes.&#42;</span></strong>: 사용된 model, input/output, token, latency, span 종류 discriminator 등 그 trace 의 핵심 데이터. §1 ~ §5 에서 본격적으로 다룸.
- <strong><span class="k">resourceAttributes.&#42;</span></strong>: <span class="k">service.name</span>, <span class="k">host.name</span>, <span class="k">host.arch</span>, <span class="k">process.pid</span>, <span class="k">process.owner</span>, <span class="k">telemetry.sdk.&#42;</span> 같은 host / process / SDK 식별 정보가 모임.
- <strong><span class="k">scope.&#42;</span></strong>: <span class="k">&#123; name, version, attributes &#125;</span> 형태. trace 를 만든 instrumentation library 식별자 (보통 name 값이 <span class="k">com.anthropic.claude_code.tracing</span>).

### Provenance (출처)

각 표의 Provenance 컬럼에 붙는 태그는 그 key 가 어디서 왔고 누가 쓰는지를 표시합니다.

### 1. Identity & session (모든 span 에 공통)

<span class="k">utils/telemetryAttributes.ts:getTelemetryAttributes()</span> 에서 만들어져 <span class="k">createSpanAttributes()</span> (<span class="k">sessionTracing.ts:258</span>) 에서 모든 span 에 merge 됩니다.

| Key | Provenance | Description |
| --- | --- | --- |
| attributes.user.id | <span class="tag tag-upstream">Upstream</span> | 머신 단위로 고정된 user 식별자. `getOrCreateUserID()` 가 홈 디렉토리의 식별자 파일에 저장한 random UUID. OAuth account id 와는 다름. |
| attributes.session.id | <span class="tag tag-upstream">Upstream</span> | 프로세스 단위 session UUID (`getSessionId()`). 프로세스를 새로 띄울 때마다 새 값. `OTEL_METRICS_INCLUDE_SESSION_ID` 로 gating (default on). interaction span 에서는 `sessionTracing.ts:329` 에서 한 번 더 명시적으로 세팅됨. |
| attributes.terminal.type | <span class="tag tag-upstream">Upstream</span> | 감지된 터미널 emulator (vscode, iTerm.app, xterm 등). `envDynamic.terminal` 출처. |
| attributes.span.type | <span class="tag tag-upstream">Upstream</span> | span 종류 discriminator. 값: interaction, llm_request, tool, tool.blocked_on_user, tool.execution, hook. `createSpanAttributes()` 에서 무조건 세팅. |

선택적 identity 필드 (해당 기능이 켜져 있을 때만 등장):

- <span class="k">attributes.app.version</span>: 빌드 버전. <span class="k">OTEL_METRICS_INCLUDE_VERSION</span> gating (default off).
- <span class="k">attributes.organization.id</span>, <span class="k">attributes.user.email</span>, <span class="k">attributes.user.account_uuid</span>, <span class="k">attributes.user.account_id</span>: Anthropic OAuth 인증된 경우만. 보통 OAuth 인증을 안 쓰면 부재.

### 3. LLM-request span attributes (span.type = "llm_request")

Claude Code 가 실제로 LLM API 를 호출할 때 만들어지는 span. 본 reference 의 대부분 key 가 여기 모입니다.

#### 3.1 Request shape (span 시작 시점)

| Key | Provenance | Description |
| --- | --- | --- |
| attributes.model | <span class="tag tag-upstream">Upstream</span> | 호출에 넘어간 model id (예: claude-haiku-4-5-20251001, gpt-4o-mini). |
| attributes.llm_request.context | <span class="tag tag-upstream">Upstream</span> | interaction span 안에 nested 면 "interaction", 아니면 "standalone". |
| attributes.speed | <span class="tag tag-upstream">Upstream</span> | `fastMode` on 이면 "fast", 아니면 "normal". |
| attributes.query_source | <span class="tag tag-upstream">Upstream</span> | 호출을 쏜 시스템 부위. 예: main_thread, subagent, generate_session_title (Langfuse 에서는 필터링됨), compaction, away_summary. |
| attributes.langfuse.span.name | <span class="tag tag-fork">Custom</span> <span class="tag tag-langfuse">Langfuse</span> | `<id>:<model>` 포맷의 span 이름. Langfuse 트리 뷰의 span 제목. |
| attributes.langfuse.is_primary | <span class="tag tag-fork">Custom</span> <span class="tag tag-langfuse">Langfuse</span> | boolean. main thread / SDK / subagent 호출은 true; utility 호출 (compaction, speculation) 은 false. primary 만 `langfuse.trace.name`/`input`/`output` 을 덮어쓸 수 있음. |
| attributes.gen_ai.request.model | <span class="tag tag-fork">Custom</span> <span class="tag tag-genai">OTel-GenAI</span> | `attributes.model` 과 같은 값을 OTel 표준 key 로. |
| attributes.gen_ai.operation.name | <span class="tag tag-fork">Custom</span> <span class="tag tag-genai">OTel-GenAI</span> | "llm_call". |
| attributes.gen_ai.agent.name, attributes.gen_ai.agent.description | <span class="tag tag-fork">Custom</span> <span class="tag tag-genai">OTel-GenAI</span> | interaction span 과 같은 값. |
| attributes.langfuse.observation.input | <span class="tag tag-fork">Custom</span> <span class="tag tag-langfuse">Langfuse</span> | **전체 conversation history** 를 LangChain chat 포맷 `[{role,content},…]` 로. **observation Input 패널** 의 표시 값. `messagesToLangChainChatFormat()` (`sessionTracing.ts`) 이 framework boilerplate 를 메시지마다 stripping 해서 만듦. |

| Key | Provenance | Description |
| --- | --- | --- |

#### 3.4 Per-turn deltas (모든 사용자에게 노출)

upstream 이 의도한 "지난 turn 이후 새로 들어온 것" 의미입니다. **전체 conversation 이 아닙니다**. 전체 conversation 은 §3.1 의 <span class="k">langfuse.observation.input</span> 에 있습니다.

| Key | Provenance | Description |
| --- | --- | --- |
| attributes.new_context | <span class="tag tag-upstream">Upstream</span> | 같은 query_source 안에서 **이전 reported span 이후 추가된** user side 메시지의 텍스트를 concat. assistant 메시지와 system reminder 는 제외. 60KB 로 truncation. |
| attributes.new_context_message_count | <span class="tag tag-upstream">Upstream</span> | new_context 에 들어간 새 user 메시지 수. |

### 4. Tool spans (span.type = "tool", tool.execution, tool.blocked_on_user)

tool 한 건 호출은 여러 span 을 만듭니다.

- tool: tool 사용 lifecycle 전체 (input → permission decision → execution → result).
- tool.blocked_on_user: user 승인 대기 시간을 담은 child span.
- tool.execution: 실제 handler 실행을 담은 child span.

| Key | Provenance | Description |
| --- | --- | --- |
| attributes.tool_name | <span class="tag tag-upstream">Upstream</span> | tool 이름 (Bash, Read 등). |

| attributes.tool_result_* | <span class="tag tag-upstream">Upstream</span> (beta) | tool result preview / length / truncation 마커. |
| attributes.new_context | <span class="tag tag-upstream">Upstream</span> (beta) | tool result 를 "new context" 로 취급 (§3.4 의 의미를 tool turn 에 그대로 적용). |
| attributes.langfuse.observation.input, .output | <span class="tag tag-fork">Custom</span> <span class="tag tag-langfuse">Langfuse</span> | Langfuse 패널용 tool input / result. |

## 빠른 안내: "Langfuse 에서 X 는 어디서 보나요?"

| 보고 싶은 것 | 어디 |
| --- | --- |
| model 에 들어간 conversation 전체 | `attributes.langfuse.observation.input` (LLM call span 의 Input 패널) |
| model 의 응답 | `attributes.langfuse.observation.output` (Output 패널) |
| system prompt | `attributes.gen_ai.system_instructions` |
| 사용 가능한 tool 스키마 | `attributes.gen_ai.tool.definitions` |
| 토큰 사용량 / 비용 | `attributes.gen_ai.usage.input_tokens`, `attributes.gen_ai.usage.output_tokens`, cache_*_tokens |
| latency 분해 | `attributes.ttft_ms`, `attributes.duration_ms`, child span duration |
| 어떤 model 이 처리했는가 | `attributes.gen_ai.request.model` |
| retry 였는가 | `attributes.attempt` > 1 |
| trace 를 시작한 user prompt | interaction span 의 `attributes.langfuse.trace.input` (또는 `OTEL_LOG_USER_PROMPTS=1` 일 때 `attributes.user_prompt`) |
| turn 단위 delta (지난 호출 이후 새로 들어온 것) | `attributes.new_context`. 전체 prompt 가 **아님**. |
