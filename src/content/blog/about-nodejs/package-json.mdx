npm 프로젝트의 모든 길은 package.json 으로 통합니다. 어떤 패키지에 의존하는지, 어떻게 실행되는지, 어떤 이름으로 npm registry 에 배포되는지. 전부 이 한 장이 결정해요. 파이썬의 pyproject.toml 과 정확히 같은 자리에 있습니다.

## 한 줄 답

<div class="callout-accent">
package.json 은 프로젝트 루트에 놓이는 한 장짜리 JSON 파일입니다. 프로젝트의 <span class="key">이름 · 버전 · 의존성 · 실행 스크립트</span> 를 담는 메타 파일이에요. 파이썬의 pyproject.toml 에 정확히 대응합니다.
</div>

npm 은 이 파일을 보고 두 방향을 판단합니다.

- <span class="key">업스트림</span>: 이 프로젝트가 어떤 패키지에 의존하는가
- <span class="key">다운스트림</span>: 어떤 이름·버전으로 배포되어 누가 받아갈 수 있는가
- 그리고 npm start 같은 실행 명령이 무엇인가

## 언제 필요한가요

모든 프로젝트가 반드시 가져야 하는 건 아니고, 세 경우 중 하나라도 해당되면 필요해요.

<div class="callout">

- <span class="key">다른 사람과 공유</span>: GitHub 에 푸시해 남이 받아 npm install 로 동일 환경을 재구성해야 할 때
- <span class="key">npm 에 공개 패키지로 배포</span>
- <span class="key">의존성 재현이 필요한 모든 상황</span>: CI/CD, Docker, 배포 서버 등

</div>

반대로, 혼자서 로컬에서 한 번 쓰고 말 임시 스크립트면 없어도 됩니다. 다만 실무에서는 거의 항상 필요해지는 파일이에요.

## npm init 으로 만들기

프로젝트 디렉토리에서 명령어 하나면 됩니다.

```bash
npm init        # 대화형 프롬프트. 여러 질문을 물어봄
npm init -y     # 기본값으로 바로 생성 (실무에서 가장 흔함)
```

-y (= yes) 는 모든 질문을 기본값으로 답해 파일을 즉시 만들어줍니다. 필드는 나중에 에디터로 직접 고치면 되고요.

대화형 모드가 물어보는 것:

<div class="callout">

- package name: 기본값은 디렉토리 이름
- version: 기본값 1.0.0
- description, entry point (index.js), test command
- git repository, keywords, author, license

</div>

## 주요 필드

최소한의 package.json 예시.

```json
{
  "name": "my-project",
  "version": "1.0.0",
  "description": "A short description",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "jest"
  },
  "dependencies": {
    "react": "^18.3.1"
  },
  "devDependencies": {
    "typescript": "^5.4.0"
  },
  "author": "Jet",
  "license": "MIT"
}
```

각 필드의 역할:

## JSON 포맷 주의점

파이썬의 TOML 이나 YAML 에 익숙한 사람에겐 JSON 규칙이 의외로 엄격하게 느껴집니다.

<div class="callout">

- <span class="key">키도 반드시 큰따옴표</span>: &#123;name: "x"&#125; X, &#123;"name": "x"&#125; O
- <span class="key">큰따옴표만 허용</span>: 작은따옴표 불가
- <span class="key">trailing comma 금지</span>: 마지막 항목 뒤에 쉼표 있으면 parse 에러
- <span class="key">주석 불가</span>: // 나 # 안 됨 (설명을 넣고 싶으면 별도 키로 우회)

</div>

<div class="callout-warm">
문법 에러가 나면 npm 이 package.json 자체를 못 읽어서 <span class="key">모든 npm 명령이 실패</span> 합니다. 수동 편집할 때 주의. VS Code 같은 에디터가 저장 시 검증해주는 편이라 평소엔 크게 고생하지 않지만, 원격 서버에서 vim 으로 고치다 망가뜨리는 일은 종종 있어요.
</div>

## 한 줄 요약

<div class="callout-accent">
package.json 은 pyproject.toml 의 JSON 버전. 프로젝트 이름·버전·의존성·실행 스크립트를 한 자리에 선언합니다. npm init -y 로 스켈레톤을 만들고, 필요한 필드를 채워가는 식으로 써요.
</div>

다음 글에서는 package.json 안의 의존성에 붙는 ^1.2.3, ~1.2.3 같은 표기. semver. 를 짧게 정리합니다. 이 두 기호의 의미만 알면 대부분의 package.json 을 읽을 수 있어요.
