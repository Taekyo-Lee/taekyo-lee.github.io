처음 Claude Code 를 깔려고 했을 때 이런 에러가 떨어졌어요. <span class="key">requires Node 22+</span>. 시스템에 깔린 Node 는 18 인데. 그렇다고 시스템 Node 를 업그레이드하자니 다른 프로젝트들이 걱정. 답은 <span class="term">nvm</span> 이었어요. 이 글은 그 도구의 핵심만 정리합니다.

## 한 줄 답

<div class="callout-accent">
<span class="term">nvm</span> (Node Version Manager) 은 여러 Node 버전을 <span class="key">한 머신에 동시 설치</span> 하고 <span class="key">셸에서 즉시 전환</span> 할 수 있게 해주는 도구예요. OS 기본 Node 는 보통 낡거나 버전 관리가 번거로운데, <span class="term">nvm</span> 으로 홈 디렉토리 (&#126;/.nvm/) 에 깔면 <span class="key">sudo 없이</span> 모든 걸 할 수 있어요. 파이썬의 <span class="term">pyenv</span> 와 완전히 같은 자리입니다.
</div>

## 왜 Node 버전 관리가 필요한가

### ① 많은 AI CLI 가 특정 Node 버전 이상을 요구

```
Claude Code      → Node 22.12.0 이상
Cursor CLI       → Node 20 이상
일부 MCP 서버    → Node 20 이상
```

OS 가 기본 제공하는 Node (예: Ubuntu 의 apt install nodejs) 는 <span class="key">몇 년 전 버전</span> 인 경우가 많아요. Claude Code 를 깔려 했는데 requires Node 22+ 에러가 떨어지는 상황. 이게 가장 흔한 첫 장애물입니다.

### ② 프로젝트마다 요구하는 Node 버전이 다를 수 있음

AI 도구 A 는 Node 22 를 요구하고, 회사 내부 프로젝트 B 는 Node 18 에 고정돼 있을 수 있어요. <span class="key">한 머신에 Node 를 하나만 깔 수 있다면 둘 중 하나는 못 돌림</span>. 이게 <span class="term">nvm</span> 같은 도구가 존재하는 이유.

### ③ 시스템 Node + sudo 지옥을 피하기

OS 가 깐 Node (/usr/bin/node) 는 시스템 디렉토리에 있어서 <span class="term">npm install -g</span> 시 sudo 가 필요해져요. 이전 글들에서 본 것처럼 <span class="key">보안상 피해야 할 패턴</span>. <span class="term">nvm</span> 으로 Node 를 <span class="key">홈 디렉토리 아래</span> (&#126;/.nvm/) 에 깔면 전역 설치도 sudo 없이 그냥 됩니다.

## <span class="term">nvm</span> 설치

공식 설치 스크립트 (최신 명령은 공식 저장소 [github.com/nvm-sh/nvm](https://github.com/nvm-sh/nvm) 에서 확인):

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
```

실행하면:

<div class="callout">

- &#126;/.nvm/ 폴더 생성
- &#126;/.bashrc 또는 &#126;/.zshrc 에 <span class="term">nvm</span> 로드 라인 자동 추가
- 새 셸을 열면 <span class="term">nvm</span> 명령을 쓸 수 있음 (또는 source &#126;/.bashrc)

</div>

설치 후 확인:

```bash
nvm --version     # 예: 0.40.0
```

## 주요 명령 5개

실전에서 99% 의 경우 이 다섯 개면 충분해요.

```bash
# Node 설치
nvm install --lts           # 최신 LTS (Long Term Support). 가장 안전한 선택
nvm install 22              # Node 22 의 최신 패치
nvm install 20.11.0         # 특정 버전 정확히

# 설치된 버전 목록
nvm ls                      # 로컬에 깔린 버전들
nvm ls-remote               # registry 에서 받을 수 있는 전체 목록 (매우 김)

# 현재 셸에서 버전 전환
nvm use 22                  # 현재 셸만 22 로 바뀜 (다른 터미널은 영향 없음)
nvm use --lts
nvm current                 # 지금 어떤 버전을 쓰고 있는지

# 기본값 설정 (새 셸이 열릴 때 자동 적용)
nvm alias default 22
nvm alias default --lts

# 제거
nvm uninstall 18            # 18 버전 제거
```

### install 과 use 의 차이

헷갈리기 쉬운 지점. 둘의 역할이 달라요.

<div class="callout-accent">

- <span class="term">nvm</span> install &lt;버전&gt;: <span class="key">다운로드 + 설치 (영구)</span>. &#126;/.nvm/versions/node/&lt;버전&gt;/ 아래에 저장
- <span class="term">nvm</span> use &lt;버전&gt;: <span class="key">현재 셸에서 어떤 버전을 쓸지 전환 (휘발성)</span>. 새 터미널을 열면 기본 버전으로 돌아감

</div>

새 버전을 처음 쓸 때는 보통 install 하면 자동으로 use 도 돼서 헷갈리지 않지만, 두 명령의 성격이 다르다는 걸 알아두세요.

## <span class="term">.nvmrc</span>: 프로젝트별 Node 버전 고정

프로젝트 루트에 <span class="key">.nvmrc 파일</span> 을 두면 "이 프로젝트는 이 Node 버전을 써야 한다" 를 선언할 수 있어요.

```bash
# <프로젝트 루트>/.nvmrc
22.12.0
```

그 폴더에서:

```bash
nvm use                # 인자 없이 호출하면 .nvmrc 를 읽어서 그 버전으로 전환
```

파이썬의 <span class="term">.python-version</span> (<span class="term">pyenv</span> 가 읽는 파일) 과 완전히 같은 개념. 팀 프로젝트에서 <span class="term">.nvmrc</span> 를 커밋하면 모든 기여자가 같은 Node 버전을 쓰게 강제할 수 있어요.

<div class="note-box">
<span class="key">자동 전환</span>. &#126;/.bashrc · &#126;/.zshrc 에 설정을 추가하면 cd &lt;프로젝트&gt; 시 자동으로 <span class="term">.nvmrc</span> 의 버전으로 스위치되게도 할 수 있습니다 (공식 문서의 "Deeper Shell Integration" 참고).
</div>

## Windows 는 어떻게?

<span class="term">nvm</span> (<span class="term">nvm</span>-sh) 은 <span class="key">Unix 전용</span> 이에요. bash 스크립트 기반이라 Windows 네이티브에선 안 돌아요. Windows 사용자는 대안을 써야 합니다.

<table class="compare-table">
  <thead>
    <tr><th>도구</th><th>특징</th></tr>
  </thead>
  <tbody>
    <tr><td><span class="term">nvm</span>-windows</td><td>이름만 비슷한 <strong>별도 프로젝트</strong>. 명령어는 유사하지만 호환성 주의 (<span class="term">.nvmrc</span> 처리 등 세부 차이)</td></tr>
    <tr><td><span class="term">fnm</span> (Fast Node Manager)</td><td>Rust 로 짠 현대적 대안. 빠르고 <span class="term">.nvmrc</span> 지원. macOS/Linux/Windows 모두 지원. 요즘 인기</td></tr>
    <tr><td>volta</td><td><span class="term">JavaScript</span> 도구 체인 전반 관리. 자동 전환 중심</td></tr>
  </tbody>
</table>

WSL2 를 쓰는 Windows 개발자라면 WSL 안의 <span class="key">Linux nvm</span> 을 쓰는 게 가장 깔끔해요. AI CLI 대부분이 Linux 환경에서 더 안정적.

## 파이썬과의 대응

<span class="term">nvm</span> 은 파이썬 버전 관리 도구들과 거의 1:1 로 대응돼요.

<table class="compare-table">
  <thead>
    <tr><th>Node</th><th><span class="term">Python</span></th></tr>
  </thead>
  <tbody>
    <tr><td><span class="term">nvm</span> install 22</td><td><span class="term">pyenv</span> install 3.12</td></tr>
    <tr><td><span class="term">nvm</span> use 22</td><td><span class="term">pyenv</span> shell 3.12</td></tr>
    <tr><td><span class="term">nvm</span> alias default 22</td><td><span class="term">pyenv</span> global 3.12</td></tr>
    <tr><td><span class="term">nvm</span> current</td><td><span class="term">pyenv</span> version</td></tr>
    <tr><td><span class="term">.nvmrc</span></td><td><span class="term">.python-version</span></td></tr>
    <tr><td>&#126;/.nvm/versions/node/</td><td>&#126;/.pyenv/versions/</td></tr>
  </tbody>
</table>

<span class="term">uv</span> 역시 <span class="term">Python</span> 버전 관리를 내장해서 (<span class="term">uv</span> python install 3.12) 비슷한 경험을 제공하고요.

<div class="callout">
<span class="key">개념 요약</span>. <span class="term">nvm</span> 과 <span class="term">pyenv</span> 는 <span class="key">같은 철학</span> (shim 방식으로 셸의 PATH 를 조작해서 특정 버전의 인터프리터를 고르게 함) 으로 만들어졌어요. 한 쪽에 익숙하면 다른 쪽은 명령어 이름만 바꾸면 됩니다.
</div>

## AI 엔지니어가 <span class="term">nvm</span> 을 써야 하는 상황

<span class="key">거의 항상 써야 해요</span>. 구체적으로:

<div class="callout-warm">

1. <span class="key">Claude Code · Codex · Cursor CLI 등 AI CLI 를 설치</span> 하려는데 Node 버전 요구사항에 걸림
2. <span class="key">Claude Code fork 를 빌드</span> 하려는데 engines 필드의 Node 버전을 맞춰야 함
3. <span class="key">회사 내부 프로젝트와 AI 도구가 서로 다른 Node 를 요구</span> 하는 상황
4. <span class="key">sudo npm install -g 실수를 피하기</span> 위한 안전한 기본 환경

</div>

처음 개발 환경을 잡을 때 <span class="key">첫 순서로 nvm 을 깔아 두면</span> 이후 모든 Node 관련 문제가 부드럽게 풀립니다. "Node 가 이미 깔려 있어도 <span class="term">nvm</span> 으로 다시 깔아 둔다" 는 게 보편적인 권장 사항.

## 한 줄 요약

<div class="callout-accent">
<span class="term">nvm</span> = 한 머신에 여러 Node 버전을 깔고 셸에서 즉시 전환. 파이썬의 <span class="term">pyenv</span> 와 완전히 같은 자리. <span class="key">프로젝트별 .nvmrc</span> 로 버전 고정, <span class="key">&#126;/.nvm/ 설치</span> 로 sudo 없이 전역 설치 가능. AI CLI 를 깔기 전에 <span class="key">가장 먼저 세팅할 도구</span> 입니다. Windows 는 <span class="term">fnm</span> 또는 WSL + Linux <span class="term">nvm</span> 을 권장.
</div>
