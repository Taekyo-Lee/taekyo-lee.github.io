이전 글에서 plugin 이 어떻게 생겼는지 (배포 박스 + 5가지 새 개념) 를 정리했어요. 이번엔 진짜 실전 — Anthropic 자신의 공식 marketplace 가 안에서 어떻게 묶여 있는지 한 번 들여다봅니다. Repo 는 [github.com/anthropics/skills](https://github.com/anthropics/skills). Anthropic 이 직접 만든 만큼 plugin 설계의 모범 사례로 참고하기 좋아요.

이 repo 는 **marketplace 형태로 publish** 돼 있어서, `/plugin marketplace add anthropics/skills` 한 줄로 marketplace 를 등록해 두면 원하는 plugin 을 Claude Code 안에서 쉽게 다운로드받을 수 있어요.

## 🎭 3 Plugins = 3 페르소나

세 묶음으로 나눈 이유는 *기술적 분류* 가 아니라 **사용자가 누구냐** 예요.

<div class="plugin-card">

### <span class="term">document-skills</span> plugin <span class="persona-tag">FOR · Office worker</span>

xlsx · docx · pptx · pdf 네 개. 일상에서 Excel·Word·PowerPoint·PDF 를 만들거나 편집할 때 자동으로 발동돼요. 흥미로운 점은 — **이 4개가 Claude.ai 의 production document 기능 (스크립트로 Word/Excel/PDF 를 만들어주는 그 기능) 을 실제로 뒷받침하는 코드** 라는 것. 단순한 데모가 아니라 진짜로 도는 코드를 그대로 공개해 둔 셈이에요. License 에 차이가 있는데 (아래 참조), 이 부분이 이유입니다.

## 🪞 눈여겨볼 메타-skill

17 개 skill 중에서도 가장 눈여겨볼 두 개는 **자기 자신을 만드는 메타-skill**. 둘 다 *example-skills* plugin 안에 들어 있어요.

<div class="meta-skill meta-skill-top">
  <div class="meta-skill-header">
    <span class="meta-skill-name">skill-creator · skill 을 만드는 skill</span>
    <span class="meta-skill-pick">⭐ Top Pick</span>
    <span class="meta-skill-plugin">in example-skills</span>
  </div>
  <div class="meta-skill-desc"><span class="key">이 marketplace 에서 한 개만 깔아본다면 단연 이거예요.</span> Claude 에게 <em>"새 skill 만들어 줘"</em> 라고 하면 이게 발동합니다. 안내하는 워크플로우는 <span class="key">draft → 테스트 케이스 → eval → iterate</span>. Anthropic 이 자기 skill 을 직접 작성·검증할 때 쓰는 절차 그대로라서, skill 을 처음 만들어 보는 사람에게 가장 좋은 출발점이에요. SKILL.md 를 어떻게 써야 잘 트리거되는지부터 description 의 trigger/skip 조건 패턴까지, 빈 페이지 앞에서 막막할 일이 없어집니다.</div>
</div>

<div class="meta-skill">
  <div class="meta-skill-header">
    <span class="meta-skill-name">mcp-builder · MCP server 를 만드는 skill</span>
    <span class="meta-skill-plugin">in example-skills</span>
  </div>
  <div class="meta-skill-desc">Python (FastMCP) · Node/TypeScript (MCP SDK) 양쪽 모두 커버합니다. 단순한 코드 generation 이 아니라 <em>"MCP server 잘 만드는 법"</em> 그 자체에 가까운 가이드예요. <span class="key">single-purpose tool 설계, 명확한 input schema, 에러 처리 컨벤션, JSON Schema 작성, LLM 친화적 description 쓰기</span> 까지 한 skill 안에 정리돼 있습니다. <em>"MCP 서버 하나 만들고 싶다"</em> 가 사실 design 작업이라는 점을 자연스럽게 드러내요.</div>
</div>

이 두 메타-skill 은 **Anthropic 이 자기 도구를 만들 때 쓰는 패턴을 그대로 노출** 한 셈이라, plugin · skill · MCP 를 직접 만들려는 사람한테는 reference 가치가 압도적이에요. 실용 skill 도 몇 개 짚어두면, `webapp-testing` 은 Playwright 으로 로컬 웹앱을 테스트하고, `theme-factory` 는 일관된 시각 테마를 적용하고, `brand-guidelines` 는 Anthropic 브랜드 가이드를 적용합니다. 자료 만들 때 쓰면 결과물이 바로 *"Anthropic 톤"* 이 돼요.

## ⚡ 직접 설치해서 보기

세 명령으로 끝.

```bash
# 1) marketplace 추가 (GitHub shorthand)
/plugin marketplace add anthropics/skills

# 2) 원하는 plugin 설치 (셋 중 골라서)
/plugin install document-skills@anthropic-agent-skills
/plugin install example-skills@anthropic-agent-skills
/plugin install claude-api@anthropic-agent-skills

# 3) 등록된 skill 확인
/skills
```

`/skills` 결과에 plugin 출신 skill 들이 `🔒 plugin-name:skill-name` 형태로 뜨면 정상이에요. (자물쇠 마크는 *"plugin 출신이라 개별 disable 안 되고 plugin 단위로만 켜고 끌 수 있다"* 의 의미. 정책 잠금 같은 게 아닙니다.) 각 skill 은 적절한 작업에서 Claude 가 자동 발동시켜요 — 명시적으로 호출할 필요 없음.
