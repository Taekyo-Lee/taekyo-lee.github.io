이 시리즈의 [#3 에서 Anthropic 의 공식 marketplace](/blog/real-world-plugin-anthropic-agent-skills/) 를 봤어요 — *"17 skills 를 페르소나별 3 plugin 으로 carve-up"* 패턴. 이번엔 정반대 모양의 marketplace 로 가 봅니다. OpenAI 가 만든 [codex marketplace](https://github.com/openai/codex-plugin-cc). **Plugin 은 단 한 개인데, 그 한 개가 plugin component type 거의 전부 (skills · commands · agents · scripts · hooks) 를 안에 박아 넣은 '풍성한 single plugin'** 의 모범 사례예요.

## 🎯 무엇을 하는 plugin 인가

이름 그대로 — OpenAI 의 자체 코딩 agent **Codex 를 Claude Code 안에서 호출할 수 있게 만든 다리**. 두 가지 큰 시나리오:

- **Code review**: `/codex:review` — 현재 작업을 OpenAI 의 reviewer 에게 보여 줌. `/codex:adversarial-review` 는 좀 더 공격적인 challenge 모드.
- **Task delegation**: `/codex:rescue` — Claude 가 막혔거나 다른 시각이 필요할 때 Codex 에 작업을 넘김. `/codex:status` · `/codex:result` · `/codex:cancel` 로 백그라운드 job 을 관리.

전제 조건: ChatGPT 구독 또는 OpenAI API 키 + Node 18.18 이상. `@openai/codex` CLI 가 시스템에 깔려 있어야 하고요. (`/codex:setup` 이 환경 점검과 자동 설치까지 도와줍니다.)

## 🧰 한 plugin 안에 거의 모든 것

Layout 패턴부터 #3 과 다릅니다. Anthropic 은 `source: "./"` 로 repo 루트를 공유했지만, OpenAI 는 plugin 을 별도 sub-directory 로 분리해 두었어요.

## 🪝 lifecycle hook 의 실전 사용

이전 [plugins 글의 trust 섹션](/blog/claude-code-plugins/#%EF%B8%8F-plugin-%EC%84%A4%EC%B9%98%EB%8A%94-%EC%8B%A0%EB%A2%B0-%EA%B2%B0%EC%A0%95%EC%9E%85%EB%8B%88%EB%8B%A4--%ED%95%9C-%EB%B2%88-%EB%A9%88%EC%B6%94%EC%84%B8%EC%9A%94) 에서 *"plugin 안에는 hook 의 셸 명령, MCP 서버, 실행 파일 같은 실제로 실행되는 코드도 함께 담길 수 있다"* 라고 했죠. codex plugin 의 `hooks.json` 이 정확히 그 산 증거예요.

```json
{
  "hooks": {
    "SessionStart": [{ "hooks": [{
      "type": "command",
      "command": "node \"${CLAUDE_PLUGIN_ROOT}/scripts/session-lifecycle-hook.mjs\" SessionStart",
      "timeout": 5
    }]}],
    "SessionEnd": [{ "hooks": [{ "type": "command", "command": "node \"${CLAUDE_PLUGIN_ROOT}/scripts/session-lifecycle-hook.mjs\" SessionEnd", "timeout": 5 }]}],
    "Stop": [{ "hooks": [{
      "type": "command",
      "command": "node \"${CLAUDE_PLUGIN_ROOT}/scripts/stop-review-gate-hook.mjs\"",
      "timeout": 900
    }]}]
  }
}
```

Claude Code 세션이 **시작될 때 / 끝날 때 / stop 신호를 받을 때** 마다 OpenAI 가 작성한 Node.js 스크립트가 **사용자 권한으로 실행됩니다** (각각 5 초, 5 초, 900 초 (15분) timeout). `${CLAUDE_PLUGIN_ROOT}` 가 plugin 의 캐시 경로로 자동 resolve 되는 것도 그대로 드러나죠.

이 plugin 을 깔면 OpenAI 가 작성한 `.mjs` 가 내 셸에서 진짜로 돕니다. Apache 2.0 으로 코드는 공개돼 있어 직접 검토 가능 — 다만 *"검토할 수 있다"* 와 *"항상 검토한다"* 사이의 거리가 plugin 신뢰 결정의 본질이라는 점은 변하지 않아요.

## ⚡ 직접 설치해서 보기

```bash
# 1) marketplace 추가
/plugin marketplace add openai/codex-plugin-cc

# 2) plugin 설치
/plugin install codex@openai-codex

# 3) 재로드
/reload-plugins

# 4) Codex CLI 환경 점검 (없으면 자동 설치 도와줌)
/codex:setup

# 5) 첫 review 돌려보기
/codex:review --background
/codex:status
/codex:result
```

설치 후 `/agents` 에 `codex:codex-rescue` 가 떠 있으면 정상. Claude 가 막힐 때 자동으로 발동돼서 Codex 에 위임할 거예요.
