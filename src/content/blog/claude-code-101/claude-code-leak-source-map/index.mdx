2026년 3월 31일, JavaScript 패키지 레지스트리 npmjs.com 에 publish 된 <span class="term">Claude Code</span> v2.1.88 패키지가 약 <span class="key">1,900 개 파일·512,000 줄의 TypeScript 원본</span>을 통째로 노출시켰습니다. 누군가 시스템에 침입해 빼낸 것이 아니라, 배포 파일 옆에 <span class="term">source map</span> **한 파일이 함께 올라간 것**이 발단이었어요. 결과적으로 원본에 가까운 코드가 **통째로 드러났습니다**.

## 📦 무슨 일이 있었나

Claude Code 는 npm 패키지로 배포됩니다.

<Image src={npmClaudeCode} alt="npmjs.com 의 @anthropic-ai/claude-code 페이지 — bin/, LICENSE.md, README.md, cli-wrapper.cjs, install.cjs, package.json, sdk-tools.d.ts 가 보이고 .map 파일은 없음" />

<p class="img-caption">현재 npmjs.com 의 <code>@anthropic-ai/claude-code</code> 페이지. 정상 상태에선 cli.js 같은 빌드 파일들만 보이고 <strong>.map 파일은 없습니다</strong>. 사고가 났던 v2.1.88 에선 이 옆에 cli.js.map 이 함께 올라가 있었어요.</p>

패키지 안에는 minify 된 단일 번들 (cli.js) 이 들어 있고, 그 옆에 cli.js.map 파일이 함께 publish 되어 있었어요. 이 .map 파일이 노출되면서 결과적으로 **원본 .ts 파일 약 1,900 개에 해당하는 TypeScript 코드가 복원 가능**해진 것이 사건의 핵심입니다 (minify 가 무엇이고 .map 한 파일로 어떻게 원본을 복원할 수 있는지는 바로 다음 섹션에서 설명합니다).

직접 원인은 두 가지가 겹친 것이었습니다. (1) .npmignore 에 .map 파일을 제외하는 항목이 누락돼 있었고, (2) 빌드 도구로 쓰이는 Bun 의 한 버그 (oven-sh/bun#28001) 가 production 모드에서도 source map 을 함께 출력시키는 상황이었어요. 둘 중 하나만 정상 작동했어도 .map 은 publish 되지 않았을 텐데, <span class="key">두 방어선이 동시에 무너졌습니다</span>. Anthropic 의 책임자 Boris Cherny 는 이를 **"plain developer error"** 로 정리했습니다.

발견자는 보안 연구자 Chaofan Shou (@Fried_rice). X 에 올린 게시물이 약 **3,200만 조회**를 기록하면서 빠르게 퍼졌어요.

<p class="img-caption">사건의 시작이 된 그 게시물. <strong>"Claude code source code has been leaked via a map file in their npm registry!"</strong> 한 줄과 함께 .map 으로부터 복원한 파일 목록이 첨부되어 있습니다.</p>
Anthropic 은 인지 후 약 <span class="key">3시간 만에</span> v2.1.88 을 unpublish 하고 v2.1.87 로 롤백했지만, 그 사이에 다운로드된 사본들이 GitHub 으로 옮겨졌고 <span class="key">한 번 공개된 코드는 다시 거둘 수 없습니다</span>. 여러 mirror 가 그 시점의 스냅샷을 보존하고 있어 현재도 접근 가능한 상태예요.

## 🗺️ 잠깐, source map 이 뭔가

JavaScript 빌드 과정을 한 줄 짚고 갈게요. 개발자가 쓴 코드는 사람이 읽을 수 있게 정리돼 있지만, 사용자에게 전달되는 배포본은 **변수명·공백·줄바꿈을 모두 줄여 압축한 형태**입니다. 이걸 <span class="term">minify</span> 라고 부릅니다. **코드의 동작은 그대로**이고, 파일 크기를 줄여 다운로드 속도를 높이기 위한 가공이에요.

그런데 minify 된 코드는 사람이 읽기 어렵습니다. 그래서 빌드 도구는 minify 결과와 함께 **원본을 다시 찾을 수 있게 해주는 지도**를 한 파일 더 만들어요. 이게 .map, 즉 source map 입니다.

## 🌪️ 그 후 며칠

Anthropic 은 GitHub 에 <span class="term">DMCA takedown</span> 신청을 보냈습니다. 다만 첫 신청이 너무 광범위해서, <span class="key">8,100 개가 넘는 무관한 레포</span>까지 함께 disable 되는 일이 벌어졌어요 (그중엔 Anthropic 자신의 공개 프로젝트의 정상 fork 도 다수 포함됐습니다). 이후 신청을 철회하고 실제 leak 사본을 담은 96 개 fork 로 범위를 좁혔지만, 정작 주목도가 낮은 mirror 들은 손도 대지 못해 "무관한 8,100 개를 끄는 동안 진짜 leak 본은 살아남았다" 는 비판이 따라왔습니다.

반대편에선 파생 프로젝트들이 폭발적으로 자랐어요. 가장 눈에 띈 건 claw-code. 누출된 코드를 OpenAI Codex 로 Python, 다시 Rust 로 옮긴 변환본인데, 며칠 만에 <span class="key">10만 별을 넘기며 GitHub 역사상 가장 빨리 성장한 레포</span>가 됐습니다. OpenClaude 같은 또 다른 fork 는 Anthropic SDK 호출 부분만 OpenAI 호환 <span class="term">shim</span> 으로 바꿔, 같은 harness 가 GPT-4o, DeepSeek, Gemini, Llama 등 200 개 이상의 모델에서도 돌게 만들었습니다. <span class="key">786 줄짜리 shim 하나로 전체 구조의 portability 가 입증된 셈이에요</span>.
