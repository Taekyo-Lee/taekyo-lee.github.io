repo 자체는 marketplace-only (`plugin.json` 한 개도 없음), 17 개 skill 위에 `marketplace.json` 한 장이 얹혀 세 묶음을 정의합니다. 이 구조 자체에 대한 풀이는 [이전 글](/blog/claude-code-plugins/) 의 예시 블록 참고.

## 🎭 3 plugin = 3 페르소나

세 묶음으로 나눈 이유는 *기술적 분류* 가 아니라 **사용자가 누구냐** 예요.

<div class="persona-h">

### document-skills <span class="persona-tag">FOR · Office worker</span>

</div>

<div class="persona-h">

### claude-api <span class="persona-tag">FOR · SDK developer</span>

</div>

Skill 단 하나, 그 자체가 plugin 이에요. Anthropic SDK 를 `import` 한 코드를 다룰 때 자동 트리거 — prompt caching · model migration (4.5 → 4.7) · 새 feature (caching, thinking, batch, files 등) 추가 같은 작업에 사용. SKILL.md 의 description 이 길고 정밀한데, *"언제 이 skill 을 쓰고 언제 안 쓸지"* 의 trigger / skip 조건을 자세히 적어두는 것도 좋은 reference 패턴이에요.

## 🪞 눈여겨볼 메타-skill

이 marketplace 에 들어 있는 skill 들 중에서도 특히 흥미로운 건 **자기 자신을 참조하는 두 개**.

<div class="meta-skill">
  <div class="meta-skill-name">skill-creator — skill 을 만드는 skill</div>
  <div class="meta-skill-desc">Claude 에게 <em>"새 skill 만들어 줘"</em> 라고 하면 이 skill 이 발동해요. <span class="key">draft 작성 → 테스트 케이스 → eval 결과 → iterate</span> 의 워크플로우를 단계별로 안내합니다. Anthropic 이 자체 skill 을 작성·검증할 때 쓰는 절차가 그대로 들어가 있어서, skill 직접 쓰려는 사람한테는 가장 좋은 출발점이에요.</div>
</div>

<div class="meta-skill">
  <div class="meta-skill-name">mcp-builder — MCP server 를 만드는 skill</div>
  <div class="meta-skill-desc">Python (FastMCP) 또는 Node/TypeScript (MCP SDK) 양쪽 모두 커버. <em>"MCP server 잘 만드는 법"</em> 의 모범답안 같은 가이드. tool 설계 원칙·에러 처리·schema 작성까지 한 skill 안에 다 정리돼 있어요.</div>
</div>

이 두 메타-skill 은 **Anthropic 이 자기 도구를 만들 때 쓰는 패턴을 그대로 노출** 한 셈이라, plugin · skill · MCP 를 직접 만들려는 사람한테는 reference 가치가 가장 높아요. 실용 skill 도 몇 개 짚어두면 — `webapp-testing` 은 Playwright 으로 로컬 웹앱 테스트, `theme-factory` 는 일관된 시각 테마 적용, `brand-guidelines` 는 Anthropic 의 브랜드 가이드 적용 — 자료 만들 때 쓰면 결과물이 바로 *"Anthropic 톤"* 이 됩니다.

## ⚡ 직접 설치해서 보기

세 명령으로 끝.

```bash
# 1) marketplace 추가 (GitHub shorthand)
/plugin marketplace add anthropics/skills

# 2) 원하는 plugin 설치 (셋 중 골라서)
/plugin install document-skills@anthropic-agent-skills
/plugin install example-skills@anthropic-agent-skills
/plugin install claude-api@anthropic-agent-skills

# 3) 등록된 skill 확인
/skills
```

`/skills` 결과에 plugin 출신 skill 들이 `🔒 plugin-name:skill-name` 형태로 (locked-by-plugin 아이콘과 함께) 뜨면 정상. 각 skill 은 적절한 작업에서 Claude 가 자동으로 발동해요 — 명시적으로 호출할 필요 없음.

## 💡 한 줄 요약

<div class="callout-warm">
이 marketplace 가 보여주는 가장 큰 design lesson: <span class="key">flat 한 skill pool 을 페르소나 단위로 carve-up</span>. 17 개 skill 을 한 폴더에 평평하게 두고, marketplace.json 의 <span class="mod">skills[]</span> 배열로 누가 어떤 묶음을 받을지 정의 — 자기 marketplace 만들 때 그대로 따라가면 되는 패턴이에요.
</div>
