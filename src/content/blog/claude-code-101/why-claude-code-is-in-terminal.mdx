처음 Claude Code 를 켜보라고 하면, 사람들의 얼굴에 당황한 기색이 잠깐 스쳐 지나갑니다.

```
$ claude
```

"이게 다야?" 화면에는 검은 바탕에 커서 하나가 깜빡이고 있을 뿐이거든요. ChatGPT 도 있고, VS Code 의 Copilot 도 있고, 심지어 Claude Code 의 VS Code extension 도 있는데, 왜 또 터미널이냐는 겁니다.

저도 처음엔 같은 생각이었어요. 그런데 며칠 써보면서 이게 불친절한 설계가 아니라는 걸 알았습니다. <span class="key">의도된 위치</span> 에요. Claude Code 가 터미널에 사는 건, agent 라는 개념이 어디까지 왔는지에 대한 답이기도 합니다.

이 글은 그 이유에 대한 이야기입니다.

## 저의 여정: GUI 에서 CLI 로

몇 달 전까지 저의 AI 코딩 워크플로우는 이렇게 생겼었습니다.

**Web.** 복잡한 알고리즘이 필요하면 ChatGPT 나 claude.ai 브라우저 탭을 열었습니다. 문제를 설명하고, 답을 복사해서 내 코드에 붙여넣고, 결과를 다시 복사해서 대화창에 돌려보냅니다. Copy, paste, copy, paste.

**IDE Extension.** 편집기 안에서 바로 대화할 수 있으면 더 편해지죠. VS Code + Copilot, 이어서 Claude Code 의 extension. 편집기 사이드바에서 파일 맥락이 자동으로 공유되고, diff 도 편집기 안에서 바로 적용됐습니다.

이 두 가지가 저에게는 GUI 시대였습니다. 그리고 그 시절 AI 의 역할은 Coding Agent 였어요. 코드를 쓰고, 고치고, 리팩터링하는 도우미.

지금 저의 주 작업 도구는 **터미널** 입니다.

```
$ claude
```

이 한 줄로 시작합니다. 키보드와 마우스로 하는 거의 모든 작업을 대행해주는 도구가 깜빡이는 커서 하나 뒤에 있어요. 역할 이름도 바뀌었습니다. 이젠 Agent Harness 라고 부릅니다.

왜 이 이동이 일어났을까요?

> Claude Code 의 VS Code extension 은 터미널 버전과 **본질적으로 같은 엔진** 입니다. 터미널이 부담스러운 사용자를 위한 GUI 래퍼일 뿐, 다른 제품이 아니에요. 그러니 이 글에서 "터미널" 이라고 말할 때, extension 을 쓰는 분들도 결국 같은 이야기입니다.

## Agent 의 역할이 바뀌면, 사는 곳도 바뀐다

AI coding agent 가 지난 몇 년간 어떻게 진화해왔는지, 저는 세 단계로 나눠 봅니다.

<EraTimeline />

### ERA 1 · 코드 조각 생성기

이 시기의 agent 는 "아이디어 생성기" 에 가까웠어요. 사람이 문제를 쪼개고, 조각을 요청하고, 결과를 조립했습니다. agent 는 내 에디터 **옆** 에 떨어져 있었어요.

### ERA 2 · Coding Agent

역할이 넓어졌습니다. 개별 코드 조각이 아니라 **프로젝트 전체** 를 함께 개발하는 파트너. 여러 파일을 동시에 이해하고, 테스트를 작성하고, 버그를 추적하기 시작했죠.

그러자 agent 가 내 코드 **옆으로** 이동하는 건 자연스러운 수순이었습니다. 편집기 안으로 들어왔어요.

### ERA 3 · Agent Harness (지금)

지금의 Claude Code 는 더 이상 '코딩' 만을 위한 agent 가 아닙니다.

- 파일을 읽고 쓰고, 커밋하고, PR 을 올리고
- 테스트를 실행하고, 빌드 결과를 해석하고
- <span class="term">npm</span>, <span class="term">pip</span>, brew 같은 패키지를 설치하고
- 외부 API 를 호출하고, 브라우저를 띄우고
- 여러 모델을 오케스트레이션하고

한마디로, <span class="key">컴퓨터 앞에 앉아 키보드와 마우스로 하는 모든 작업의 대행자</span> 입니다.

이런 agent 는 편집기 **안** 에만 머물 수 없습니다. 편집기는 "코드를 쓰는 공간" 으로 설계된 곳이니까요. Agent Harness 는 <span class="key">컴퓨터 전체</span> 를 무대로 써야 합니다. 그래서 터미널로 내려왔습니다.

## 근접 모델: agent 는 컴퓨터와 가까울수록 강해진다

세 시대를 한 장의 그림으로 포개보면 명확해져요.

<LayerStack />

위에서 아래로 갈수록 **컴퓨터 자체** 에 가까워집니다. 터미널은 운영체제 바로 위의 층이에요. 파일시스템, 프로세스, 네트워크, 셸 환경. 이 모든 것에 한 계단만 내려가면 닿습니다.

Web 은 멀어요. 브라우저라는 샌드박스 안에 갇혀 있습니다.
IDE 는 한 걸음 가깝지만, 여전히 "편집기의 세계관" 안에 있어요.
Terminal 은 <span class="key">바로 위의 층</span> 입니다.

그래서 agent 가 Terminal 에 있을 때, <span class="key">할 수 있는 일의 상한이 가장 높습니다.</span> 컴퓨터로 할 수 있는 모든 일에 닿을 수 있거든요.
