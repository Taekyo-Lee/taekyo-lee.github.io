대부분의 사람들에게 AI 와의 첫 만남은 이런 모습이었을 거예요. 깔끔한 흰 배경, 가운데 자리잡은 입력창, 친근한 인사말.

<Image src={chatgptWeb} alt="ChatGPT 웹 화면 — 가운데 입력창과 What's on the agenda today? 인사말" class="web-ui-shot" />

요즘 <span class="term">Claude Code</span> 많이들 쓰시죠. 그런데 막상 켜보면 좀 다른 곳에 있습니다. 흰 배경의 채팅창이 아니라, **까만 터미널**이에요. 저도 처음엔 살짝 당황했습니다. 그런데 며칠 써보면서 이게 불친절한 설계가 아니라는 걸 알았어요. **의도된 위치**였습니다. Claude Code 가 터미널에 사는 건, **agent 라는 개념이 어디까지 왔는지**에 대한 답이기도 했어요.

이 글은 그 이유에 대한 이야기입니다.

## 🛤️ 저의 여정: GUI 에서 CLI 로

저는 매일 코드를 짜는 sw engineer 이고, 그 일의 상당 부분을 AI 와 함께 합니다. 다만 "어떻게 함께 하는지" 는 지난 몇 달 사이 꽤 빠르게 바뀌었어요.

**Web.** 처음엔 ChatGPT 와 claude.ai 였습니다. 복잡한 알고리즘이 필요하면 브라우저 탭을 열고, 문제를 설명하고, 답을 복사해서 내 코드에 붙여넣고, 결과를 다시 복사해서 대화창에 돌려보냅니다. Copy, paste, copy, paste.

**IDE Extension.** 그 왕복이 귀찮아서 편집기 안으로 들어왔습니다. VS Code + Copilot 으로 옮겨갔고, 이어서 Claude Code 의 VS Code extension 까지 써봤습니다. 편집기 사이드바에서 파일 맥락이 자동으로 공유되고, diff 도 편집기 안에서 바로 적용됐어요.

이 두 단계가 저에게는 **GUI 시대**였습니다. 그리고 그 시절 AI 의 역할은 <span class="term">Coding Agent</span> 였어요. 코드를 쓰고, 고치고, 리팩터링하는 도우미.

지금 저의 주 작업 도구는 **터미널** 입니다. 키보드와 마우스로 하는 거의 모든 작업을 대행해주는 도구가 그 안에 있어요. 역할 이름도 바뀌었습니다. 이젠 <span class="term">Agent Harness</span> 라고 부릅니다.

막상 한 번 Claude Code 를 켜보면 web UI 에 익숙한 눈으론 살짝 당황하게 됩니다. 터미널을 열고

```
$ claude
```

라고 치면, 다음과 같은 <span class="term">TUI (Terminal User Interface)</span> 가 뜨거든요.

<Image src={claudeStartup} alt="터미널에서 claude 를 실행한 직후 뜨는 Claude Code TUI 화면" />

"이게 다야?" 익숙한 채팅 UI 도, 사이드바도, 버튼도 없고, 까만 터미널 안의 작은 입력 박스 하나. AI 코딩 도구가 이미 많은데, **왜 또 CLI 냐**는 겁니다.

## 🏠 Agent 의 역할이 바뀌면, 사는 곳도 바뀐다

AI agent 가 지난 몇 년간 어떻게 진화해왔는지, 저는 **세 단계**로 나눠 봅니다.

<EraTimeline />

### ERA 1 · Coding <span class="term">Chatbot</span>

ChatGPT 가 막 나오던 시기의 모습입니다. AI 는 브라우저 탭 안에만 살았고, 내 코드베이스에는 전혀 닿지 못했어요. 사람이 문제를 머릿속에서 잘게 쪼개서, 한 조각씩 챗에 물어보고, 답을 복사해서 자기 손으로 코드에 끼워 넣었습니다. AI 는 도우미였지만, **내 에디터 옆** 에 떨어져 있는 별도의 창이었어요.

### ERA 2 · Coding <span class="term">Agent</span>

역할이 한 단계 넓어졌습니다. 짧은 답을 던져주는 도구가 아니라, **프로젝트 전체** 를 함께 개발하는 파트너가 됐어요. 여러 파일을 한 번에 읽고, 테스트를 짜주고, 막힌 버그를 같이 파고듭니다.

이 정도 일을 하려면 내 코드와 가까이 있어야 했어요. 그래서 agent 는 브라우저 탭에서 나와 **편집기 안** 으로 들어왔습니다.

### ERA 3 · Agent <span class="term">Harness</span> (지금)

이 단계에 와서는 더 이상 '코딩' 만 하지 않습니다.

- 영수증 PDF 들을 모아서 엑셀로 정리하고
- PPT 자료의 초안을 잡아주고
- 메일함을 분류해서 답장 초안까지 만들고
- 코드를 짜고, 테스트를 돌리고, 커밋하고 PR 까지 올리고
- 외부 API 호출, 브라우저 자동화, 여러 모델 오케스트레이션까지

한마디로 **컴퓨터 앞에 앉아 키보드와 마우스로 하는 거의 모든 작업** 의 대행자가 된 거예요. 더 이상 **개발자만의 도구가 아닙니다**.

이런 일을 하려면 편집기 안에만 머물 수 없습니다. 편집기는 "코드를 쓰는 공간" 으로 만들어진 곳이니까요. 무대가 컴퓨터 전체로 넓어졌고, 그래서 agent 는 **터미널까지 내려왔습니다**.

앞에서 비워뒀던 자리에 이름이 채워집니다.

## 🎯 쉽게 생각하세요. <span class="key">"agent 는 컴퓨터와 가까울수록 강해진다"</span>

세 시대를 한 장의 그림으로 포개보면 명확해져요.

<LayerStack />

위에서 아래로 갈수록 **운영체제** 와 가까워집니다. 터미널은 OS 바로 위의 층이에요. 파일시스템, 프로세스, 네트워크, 셸 환경. 이 모든 것에 한 계단만 내려가면 닿습니다.

Web 은 멀어요. 브라우저라는 샌드박스 안에 갇혀 있죠.
IDE 는 한 걸음 가깝지만, 여전히 "편집기의 세계관" 안이고요.
Terminal 은 그 다음, OS 바로 위에 있습니다.

그래서 agent 가 Terminal 에 있을 때 **할 수 있는 일이 가장 많아집니다**. 컴퓨터로 할 수 있는 거의 모든 일에 닿을 수 있거든요.
