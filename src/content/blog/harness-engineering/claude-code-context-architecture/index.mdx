## 익숙한 baseline: ReAct agent (OpenAI-style)

Claude Code 의 모양을 보기 전에, 더 익숙한 baseline 을 한번 깔고 갑니다. 2022 년 Yao 등이 제안한 <span class="term">**[ReAct agent](https://arxiv.org/abs/2210.03629)**</span> 는 거의 모든 현대 LLM agent 의 **기본 골격**이에요.

<div class="react-figure rf-narrow">

  <h3 class="rf-title">ReAct pattern</h3>
  <p class="rf-sub">Yao et al. (2022) — the foundation every modern LLM agent builds on</p>

  <div class="react-flow">
    <span class="loop-step">Thought</span>
    <span class="loop-arrow">→</span>
    <span class="loop-step">Action</span>
    <span class="loop-arrow">→</span>
    <span class="loop-step">Observation</span>
    <span class="loop-arrow repeat">↻</span>
  </div>

  <div class="rf-notes">
    <div class="rf-note react-note">
      <span class="rf-note-label">ReAct loop</span>
      LLM 이 <strong>생각 (Thought)</strong> 하고 → 도구를 <strong>호출 (Action)</strong> 하고 → 결과를 <strong>관찰 (Observation)</strong> 한 뒤 다시 생각하는 루프. 거의 모든 현대 LLM agent 의 기본 골격.
    </div>

</div>

<span class="note-flag">📝 NOTE</span> <strong>왜 *tools* 는 *messages* 와 별개의 top-level key 일까?</strong> 위 그림에서 *tools* 가 *messages* 와 별개로 분리돼 있는 게 보이죠.

직관적으로 보면 tool 정의도 그냥 messages 안의 한 메시지로 들어가도 될 것 같고, 실제로 위 빙산의 **Layer 4 (chat template)** 에 도달하면 tools 도 prompt 텍스트 안에 그냥 합쳐져 들어갑니다. 모델 입장에서는 그 시점부터 텍스트의 일부일 뿐, *별도 특별취급은 없어요*.

그래도 API 형식 차원에서 tools 를 별도 자리에 두는 이유는, 결국 **API 편의** 예요. 호출을 구성·검증·문서화하기 깔끔하고, 그 위에서 chat template 이 일관된 형식으로 prompt 를 조립할 수 있어요.

그 일관된 형식이 또 마침 모델이 post-training 단계에서 학습한 *tool-calling 전용 template* 과 정확히 맞물려서, 결과적으로 *Action* 단계의 <span class="k">tool_calls</span> 출력이 안정적으로 나오는 거고요.

## 본 주제: Claude Code agent 의 context structure

<div class="note-box">

<span class="note-flag">📝 NOTE</span> Claude Code 가 실제로 호출하는 API 는 **Anthropic Messages API** 의 형식을 따르기 때문에 OpenAI-compatible 을 100% 따르는 건 아니에요. 다만 top-level key 의 구조 (system / tools / messages) 가 사실상 동일하고 prompt assembly 흐름도 같은 모양이라, 이 글에서는 baseline 과 비교가 쉽도록 *OpenAI 스타일* 시각화를 그대로 가져와서 그립니다.

</div>

Claude Code 는 baseline 으로 본 ReAct agent 의 <span class="key">**확장**</span>이에요. **두 가지 차이**가 있습니다.

1. **System 이 별도 top-level key 로 분리**됩니다. ReAct 에서는 Messages 배열 안의 한 element (<span class="k">role: "system"</span>) 였던 system 메시지가, Claude Code 에선 top-level key 로 승격해요.
2. **각 key 안의 내용이 훨씬 풍부**해집니다.
    - System 안에 정적·동적 블록 합쳐 **20개 가까운 sub-block** 이 들어가요.
    - Tools 가 *Core primitives* (Read, Edit, Bash 등) · *Meta tools* (Skill, Agent) · *MCP tools* (<span class="k">.mcp.json</span>) 의 **세 카테고리**로 분류돼요.
    - Messages 에는 매 turn 맨 앞에 **합성 <span class="k">&lt;system-reminder&gt;</span> 채널** 이 prepend 됩니다. CLAUDE.md · MEMORY.md · 오늘 날짜 같은 컨텍스트가 여기로 흘러 들어와요.

그래서 한 번 LLM 을 호출할 때 LLM 에게 들어가는 API block 형태의 context 는 다음 <span class="key">**세 Block**</span> (한 JSON 객체의 세 top-level key) 으로 구성돼요.

<div class="three-siblings">
  <div class="sk sk-system">
    <div class="sk-name">System</div>
    <div class="sk-desc">Claude Code 가 매 turn 깔아 두는 시스템 프롬프트. 정적 + 동적 블록.</div>
  </div>
  <div class="sk sk-tools">
    <div class="sk-name">Tools</div>
    <div class="sk-desc">model 에 노출된 모든 도구의 schema. Core / Meta / MCP 세 카테고리.</div>
  </div>
  <div class="sk sk-messages">
    <div class="sk-name">Messages</div>
    <div class="sk-desc">합성 reminder + conversation history + 이번 차례의 user 입력.</div>
  </div>
</div>

세 key 는 한 JSON 객체 안에서 <span class="key">**동등한 위치에 놓인 독립 채널**</span> 입니다. Langfuse 같은 tracing 도구에서도 *System Instructions* · *Tools* · *Input* 세 패널로 따로따로 분리되어 보여요.

이 시리즈에선 세 채널을 각각 <span class="term">System Block</span> / <span class="term">Tools Block</span> / <span class="term">Messages Block</span> 이라 부릅니다. API key 의 단/복수 (system / tools / messages) 를 그대로 보존하면서, 시리즈가 정의한 고유 영역임을 표시하는 capitalized 형태예요. 후속 글에서 이 이름으로 cross-reference 합니다.

## 작은 surprise 하나

이 anatomy 를 처음 보는 사람들이 가장 의외로 느끼는 건 보통 <span class="key">**4번 영역**</span> 입니다.

<div class="surprise">
<span class="label">⚡ SURPRISE</span> *"내가 쓴 **CLAUDE.md** 는 당연히 system prompt 안에 들어가는 거 아니야?"* 라고 생각하기 쉽죠. 그런데 실제로는 그렇지 않아요.

**CLAUDE.md · MEMORY.md · 오늘 날짜** 처럼 harness 가 끌어오는 컨텍스트는 system prompt 가 아니라, Messages 배열의 **첫 번째 user message** 에 합성으로 prepend 되는 <span class="k">&lt;system-reminder&gt;</span> 블록 안에 들어갑니다. system prompt 자체엔 Anthropic 이 정의한 정적·동적 블록만 담기고, *Anthropic 바깥에서* 들어오는 컨텍스트는 별도 채널 (4번 영역) 로 흐르는 거죠.
</div>


## Take-home

이 글에서 챙겨갈 정리:

> Claude Code 가 LLM 에 보내는 한 번의 입력은 <span class="key">**세 Block**</span> (System · Tools · Messages) 으로 나뉘고, 그 안에 총 <span class="key">**6 영역**</span> 이 채워져 있어요.
>
> - System: 정적 system · 동적 system
> - Tools: tool 정의들
> - Messages: 합성 reminder · conversation history · 이번 입력
